# IBM Racing League — TORCS Autopilot | Vinci Eco Drive

## Our Team: Vinci Eco Drive
Vinci Eco Drive is a student association from Pôle Léonard de Vinci in Courbevoie, France. 

## What is the IBM Racing League?
It is a competition where participants develop autonomous driving agents for the TORCS simulator. The Python client communicates with the simulator in real-time to read sensor data and send back driving commands (steering, throttle, braking, gear).

## Project Files
* `autopilot.py`: The autonomous driving agent (the brain of the car).
* `report.py`: The post-race telemetry analyzer.

## Prerequisites
* Windows OS, or Mac OS using Wine.
* Python 3.8+ installed on your system.
* Python packages for the report: pandas, numpy, matplotlib
* TORCS package : https://ibm.biz/TORCSdownloadzip

## Setup & Launch

### 1. Installation
* Extract the downloaded archive into the `C:\\torcs\\` directory.

### 2. Simulator Configuration
* Navigate to `C:\\torcs\\torcs\\` and run `wtorcs.exe`.
* In the TORCS menu, select the following sequence: `Race` > `Quick Race` > `Configure Race`.
* Choose the track of your choice.
* In the driver selection, you **must** select `scr_server 1`.

### 3. Starting the Autopilot
* Run the `autopilot.py` script before the race starts.
* Once the UDP connection is established, the car will automatically take control.

## Telemetry & Analysis
When the race ends (or the simulator is closed), the autopilot automatically saves two files in the `report/` folder:
1.  **`telemetry.csv`**: A full log of sensor readings and control outputs (50 rows per second).
2.  **`report.html`**: An interactive HTML report generated by `report.py`, featuring charts for speed traces, braking points, G-forces, and more.
